# PRD: 채용 플랫폼 메인 페이지

- 문서 버전: v2.0
- 작성일: 2026-07-13
- 대상 화면: 메인 페이지 (GNB + 검색 + 지역 필터 + 정렬 + 채용 공고 리스트)

---

## 1. 핵심 기능 목록 (Feature List & Data Fields)

### 1.1 GNB (최상단 내비게이션)

| 필드명 | 타입 | 설명 | 필수 |
|---|---|---|---|
| `tab_id` | string | 탭 고유 식별자 (job, resume, event, community) | Y |
| `label` | string | 탭 표시명 (채용/이력서/고용이벤트/커뮤니티) | Y |
| `route_path` | string | 탭 클릭 시 이동 경로 | Y |
| `order` | number | 탭 노출 순서 | Y |
| `is_active` | boolean | 현재 페이지 여부 (활성 스타일 표시) | Y |
| `badge_count` | number | 신규 알림/공지 뱃지 (커뮤니티 등) | N |

**동작 요구사항**: 활성 탭은 하단 인디케이터 또는 폰트 강조로 구분. 탭 순서/노출은 CMS에서 제어 가능하도록 설계 권장.

---

### 1.2 검색 섹션

| 필드명 | 타입 | 설명 | 필수 |
|---|---|---|---|
| `keyword` | string | 사용자가 입력한 검색어 | Y |
| `placeholder_text` | string | 입력 전 안내 문구 (예: "직무, 기업명, 기술스택으로 검색") | Y |
| `recent_searches` | string[] | 최근 검색어 목록 (로컬/서버 저장) | N |
| `autocomplete_suggestions` | string[] | 실시간 자동완성 후보 | N |
| `search_scope` | enum [전체, 공고, 기업] | 검색 범위 지정 (선택 시) | N |

**동작 요구사항**: 검색 시 지역 필터/정렬 조건과 AND 조건으로 결합되어 리스트에 즉시 반영.

### 1.3 지역 필터 (2단 계층형)

| 필드명 | 타입 | 설명 | 필수 |
|---|---|---|---|
| `region_l1_id` | string | 시/도 코드 | Y |
| `region_l1_name` | string | 시/도명 (서울, 경기 등) | Y |
| `region_l2_id` | string | 시/군/구 코드 | Y |
| `region_l2_name` | string | 시/군/구명 (강남구, 수원시 등) | Y |
| `parent_l1_id` | string | 시/군/구가 속한 상위 시/도 참조키 | Y |
| `selected_regions` | array | 다중 선택된 지역 목록 | N |
| `is_all_selected` | boolean | "전체" 옵션 선택 여부 | N |

**동작 요구사항**: 시/도 선택 시 하위 시/군/구 목록 갱신, 다중 선택 가능, 선택된 지역은 칩(chip) 형태로 상단에 노출 및 개별 해제 가능.

### 1.6 카드 Hover 인터랙션

| 항목 | 스펙 |
|---|---|
| 트리거 | `mouseenter` / `mouseleave` (키보드 `focus`/`blur`도 동일 적용 — 접근성) |
| 확대 효과 | `transform: scale(1.03)`, `transition: transform 200~250ms ease-out` |
| 그림자 | `box-shadow` 강화 (예: 기본 `0 1px 3px rgba(0,0,0,.1)` → hover `0 8px 24px rgba(0,0,0,.2)`) |
| 레이어 분리 | hover 카드 `z-index` 상향 (기본 1 → hover 10) |
| 오버레이 레이어 | 반투명 배경(`rgba(0,0,0,0.5)` 등) + `backdrop-filter: blur(8px)`, `opacity 0→1` 페이드인 (250ms) |
| 상세 조건 텍스트 | 경력 / 학력 / 급여 / 직급 / 근무일수 / 근무지역 — 고대비 색상(예: 흰색 텍스트, WCAG AA 대비율 4.5:1 이상) |
| 접근성/예외 | 모바일·터치 디바이스는 hover 미지원 → 탭(tap) 시 상세 오버레이 노출 또는 상시 요약 정보 표시로 대체 |

## 2. 사용자 시나리오

### 시나리오 1 — 지역 기반 탐색
- **사용자**: 특정 지역 내 이직을 희망하는 경력 구직자
- **목표**: 거주지 인근 채용 공고만 빠르게 필터링
- **흐름**: 메인 진입 → 좌측 지역 필터에서 "경기"(시/도) 선택 → 하위 "수원시"(시/군/구) 선택 → 리스트 자동 갱신 → 기본 정렬(최신순)으로 공고 확인
- **기대 결과**: 3클릭 이내로 원하는 지역의 공고만 노출, 선택 지역 칩으로 현재 필터 상태 인지 가능

### 시나리오 2 — 키워드 검색 + 정렬 변경
- **사용자**: 특정 직무(예: "백엔드 개발자")를 찾는 구직자
- **목표**: 키워드로 좁힌 뒤 인기 있는 공고 우선 확인
- **흐름**: 검색창에 "백엔드 개발자" 입력 → 결과 리스트 갱신 → 우측 상단 정렬을 "인기순"으로 변경 → 지원자 수/조회수 높은 공고 상단 노출
- **기대 결과**: 검색+정렬 조합이 즉시(비동기) 반영, 로딩 지연 시 스켈레톤 UI 제공

### 시나리오 3 — 카드 Hover로 빠른 조건 확인
- **사용자**: 다수 공고를 훑어보며 조건을 빠르게 비교하는 구직자
- **목표**: 공고 상세 페이지 진입 없이 핵심 조건만 빠르게 스캔
- **흐름**: 리스트 카드 위에 마우스 오버 → 카드 확대 + 그림자 강조 → 블러 오버레이 페이드인 → 경력/학력/급여/직급/근무일수/근무지역 텍스트 확인 → 조건 불일치 시 마우스 이동, 일치 시 클릭하여 상세 진입
- **기대 결과**: 페이지 전환 없이 평균 탐색 시간 단축, 오버레이 정보만으로 1차 필터링 가능

### 시나리오 4 — 모바일/터치 환경 대응
- **사용자**: 모바일로 접속한 구직자
- **목표**: Hover가 불가능한 환경에서도 동일한 상세 조건 확인
- **흐름**: 카드 탭(1차) → 오버레이 표시 → 카드 재탭(2차) 또는 별도 버튼으로 상세 페이지 이동
- **기대 결과**: 데스크톱과 동등한 정보 접근성 제공, 오조작(실수 클릭) 방지

## 3. 개발 우선순위

| 우선순위 | 범위 | 포함 기능 | 근거 |
|---|---|---|---|
| **P0 (MVP)** | 핵심 탐색 흐름 | GNB(탭 라우팅), 검색(키워드 입력/조회), 채용 공고 리스트(카드 기본 정보 노출), 정렬(최신순 기본 제공) | 서비스의 최소 사용성(탐색→결과 확인)을 보장하는 필수 골격. 별도 산출 로직 없이 구현 가능 |
| **P1** | 필터/정렬 고도화 | 2단 계층형 지역 필터(다중 선택, 칩 UI), 정렬 옵션 확장(추천순/인기순 — 백엔드 스코어링 필요) | 사용자 만족도에 직접적 영향을 주지만, 인기순/추천순은 조회수·지원수 집계 및 가중치 로직 등 백엔드 선행 작업 필요 |
| **P2** | 인터랙션/비주얼 폴리싱 | 카드 Hover 확대·그림자·z-index, 블러 오버레이 페이드인, 고대비 상세조건 텍스트, 모바일 터치 대응 | 데이터 정합성과 무관한 시각적 완성도 요소. 핵심 기능(P0/P1) 안정화 이후 애니메이션 성능·접근성 검증과 함께 적용하는 것이 리스크가 낮음 |
| **P3 (후속)** | 개인화/편의 기능 | 최근 검색어, 자동완성, 북마크(스크랩), 검색 범위 세분화 | 리텐션 향상 목적의 부가 기능으로 초기 출시 이후 사용 데이터를 기반으로 우선순위 재산정 권장 |

**우선순위 판단 기준**: (1) 핵심 사용자 여정(탐색→비교→지원) 완성 여부, (2) 백엔드 의존도(스코어링 로직 등)가 높은 기능은 후순위 배치, (3) 시각 효과는 기능 완성 후 성능/접근성 검증을 거쳐 적용.
