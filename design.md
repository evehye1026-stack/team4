# SHIPYARD Careers — Design Spec

> 개발 직군 **전용** 채용 사이트.
> 참고 레퍼런스(네이버 커리어스)의 레이아웃 감성을 계승하되, IP는 사용하지 않고 독자 브랜드로 재구성한다.

---

## 1. 컨셉

- **한 문장:** "코드로 세상을 ship 하는 사람들을 위한 채용 사이트."
- **오직 개발 직군만** 노출·검색·지원 가능. 비개발 직군(디자인·기획·마케팅 등)은 이 사이트의 범위가 아니다.
- 스페이스/조선소(shipyard) 은유 = 배를 짓는 곳 + 코드를 배포(ship)하는 곳. 히어로의 우주 무드를 브랜드 서사로 연결한다.
- 톤: 조용하고 정제된 화이트 스페이스 + 단 한 곳(히어로)에서만 대담하게.

---

## 2. 브랜드 & 카피

- **브랜드명:** SHIPYARD (플레이스홀더 — 실제 사명으로 교체 가능)
- **히어로 헤드라인:** `Ship what's next.`
- **섹션 타이틀:** `Roles` (레퍼런스의 "Teams" 자리)
- **서브카피:** 서로 다른 코드가 모여 하나의 제품이 됩니다
- 카피 원칙: 시스템 용어가 아니라 사용자가 아는 말로. 버튼은 하는 일 그대로 ("검색", "지원하기").

---

## 3. 컬러 토큰

| 토큰 | HEX | 용도 |
|------|-----|------|
| `--ink` | `#0B1020` | 본문/헤드라인/블랙 버튼 |
| `--bg` | `#FFFFFF` | 페이지 배경 |
| `--paper` | `#F6F7FB` | 섹션 대비 배경 |
| `--line` | `#E6E8EF` | 헤어라인 구분선 |
| `--hero-deep` | `#070B24` | 히어로 그라디언트 하단 |
| `--hero-navy` | `#101A54` | 히어로 그라디언트 중간 |
| `--accent` | `#6C4DF6` | 보라 포인트(배너/링크/호버) |
| `--accent-glow` | `#8E7BFF` | 히어로 앰비언트/포커스 |
| `--signal` | `#4DE3D0` | 개발 시그니처(카운트·터미널 커서) |

- 화이트 + 딥네이비 + 바이올렛은 레퍼런스 정체성 계승.
- `--signal`(민트/시안)은 터미널·시그널 느낌으로 "개발 전용"임을 은근히 드러내는 액센트.

---

## 4. 타이포그래피

| 역할 | 폰트 | 이유 |
|------|------|------|
| Display | **Space Grotesk** | 기하학적 그로테스크. 스페이스/테크 무드에 맞고 흔한 serif 디폴트를 피함 |
| Body | **Inter** | 가독성 높은 UI 본문 |
| Mono | **JetBrains Mono** | 카운트·필터·라벨. "코드" 시그니처 |

- 타입 스케일: Hero 클램프 `clamp(44px, 9vw, 128px)` / Section `clamp(40px, 6vw, 72px)` / Role `clamp(26px, 3vw, 40px)` / Body `16–18px` / Mono label `12–13px, letter-spacing .08em`.
- 카운트와 필터 라벨은 **모노 + 대문자**로 처리해 개발 정체성을 부여(시그니처).

├───────────────────────────────────────────────┤
│                                                 │
│                 (우주 그라디언트 + 앰비언트)        │  ← 히어로

│   └───────────────────────────────────────┘    │
├───────────────────────────────────────────────┤
│  Roles                                          │
│  서로 다른 코드가 모여…            Learn more →    │
│                                                 │
│  Frontend        42 ↗   사용자와 맞닿은 화면을 …   │
│  ──────────────────────────────────────────    │
│  Backend         51 ↗   보이지 않는 힘을 설계…     │
│  ──────────────────────────────────────────    │
│  … (개발 직군만)                                  │

---

## 6. 컴포넌트

### 6.1 검색/필터 바
- `키워드` 텍스트 인풋 · `조직` 셀렉트 · `직군` 셀렉트 · 블랙 `검색` 버튼.
- **`직군` 셀렉트에는 개발 직군만** 들어간다. 비개발 직군 옵션 자체가 없다.
- 우측에 `개발 직군만 채용` 배지로 범위를 명시.

### 6.2 Role 리스트 아이템
- 좌: 직군명(Display) + 카운트(Mono) + `↗`.
- 우: 한국어 태그라인.
- 호버 시 라인이 `--accent`로, `↗`가 우상향 이동. 클릭 시 하단 포지션 패널 필터링.

### 6.3 포지션 카드
- 직무명 · 조직 · 근무형태 · 스택 태그 · `지원하기` 버튼.
- 모든 카드는 개발 직무 데이터만. `지원하기`는 지원 플로우 진입 지점.

---

## 7. 개발 직군 데이터 (초기값)

| 직군 | 태그라인 | 카운트(예시) |
|------|----------|------|
| Frontend | 사용자와 맞닿은 화면을 만듭니다 | 42 |
| Backend | 보이지 않는 힘을 설계합니다 | 51 |
| Mobile | 손안의 경험을 빚습니다 | 18 |
| AI · ML | 지능을 코드로 옮깁니다 | 27 |
| Infra · DevOps | 흔들리지 않는 토대를 세웁니다 | 23 |
| Data | 데이터에서 답을 캐냅니다 | 16 |
| Security | 신뢰의 마지막 방어선을 지킵니다 | 9 |

---

## 8. 모션 (절제)

- 히어로: 페이지 로드 시 헤드라인 마스크 업 + 앰비언트 파티클/도형 아주 느린 부유.
- 리스트: 스크롤 진입 시 순차 페이드업(짧게).
- 마이크로: 링크·카드 호버 트랜지션 150–200ms.
- `prefers-reduced-motion: reduce` 존중 — 모든 부유/시퀀스 제거.

---

## 9. 반응형 / 접근성 (품질 하한선)

- 모바일(≤640px)에서 필터 바 세로 스택, 네비 축약.
- 키보드 포커스 링 상시 노출(`--accent-glow`).
- 대비 AA 이상(블랙 텍스트/화이트, 화이트 텍스트/딥네이비).
- 시맨틱 마크업 + `aria-label`. 셀렉트/버튼 네이티브 요소 사용.

---

## 10. 교체 포인트 (실제 도입 시)

- 브랜드명 · 로고 · 히어로 3D 에셋
- 조직(법인) 목록, 직군 카운트, 포지션 실데이터 연동(API)
- `지원하기` → 실제 ATS/지원 폼 라우팅
